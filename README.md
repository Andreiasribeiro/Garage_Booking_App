# Garage Booking System Web App (AR Moto Garage)

Full‑stack web application developed for a small garage. Customers can register and book services; the owner manages bookings, generates invoices, updates services, and assigns staff.

**Technologies**: HTML5, CSS3, JavaScript, React.js, Node.js, Express, MongoDB, Expo (for mobile prototyping)

---

## 🔧 Features

- **User side**: registration, service booking, view booking status
- **Admin dashboard**: manage bookings, add/edit services, allocate staff, generate invoices

---

## 🧪 API Testing 

During development, I used **Postman** to test all backend endpoints. This included:

- Setting up a local environment
- Understanding the database schema (MongoDB collections)
- Testing CRUD operations for bookings, users, services, and invoices
- Validating authentication and error handling

---

## 🚀 How to Run Locally

1. Clone the repository: `git clone https://github.com/your-username/Garage_Booking_App.git`
2. Install backend dependencies: `cd server && npm install`
3. Install frontend dependencies: `cd client && npm install`
4. Set up MongoDB (local or Atlas) and add your connection string in a `.env` file inside `/server`
5. Start the backend: `npm run server` (from `/server`)
6. Start the frontend: `npm start` (from `/client`)
